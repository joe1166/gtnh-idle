# GTNH Idle 多语言（给策划看的）

> 目标：你只需要维护一个表格 `tables/locale.csv`，把空着的翻译补齐即可。
> 程序在开发模式运行游戏时，会自动把“缺的 key”补进这个 CSV。

---

## 1. 你需要改哪个文件？

- **唯一需要你维护的文件**：`tables/locale.csv`

CSV 结构是：

| 列名 | 含义 | 你要做什么 |
|---|---|---|
| `key` | 文案 ID（不要改） | 不要改、不排序也行 |
| `zh` | 中文 | 填中文（可留空） |
| `en` | 英文 | 填英文（可留空） |

提示：如果 `zh/en` 为空，就表示“这条翻译还没写”。

---

## 2. 怎么知道哪些文案没写？

用 Excel 打开 `tables/locale.csv` 后：

1. 对 `zh` 或 `en` 列筛选空白（筛选 → 空白）
2. 这些空白就是待补翻译的清单

---

## 3. Excel 打开/保存 CSV 的注意事项（避免乱码）

### 打开
直接双击 `tables/locale.csv` 打开即可（文件已带 UTF-8 BOM，正常不会乱码）。

### 保存（关键）
Excel 保存 CSV 时要选 **“CSV UTF-8（逗号分隔）(*.csv)”**。

- 不要选普通 “CSV（逗号分隔）”，否则可能变成 ANSI 导致中文乱码。

如果你的 Excel 没有这个选项：
- 建议用“数据 → 自文本/CSV 导入”，编码选择 UTF-8，然后再另存为 “CSV UTF-8”。

---

## 4. 新功能的文案怎么进表？（你不需要自己找）

程序同学在开发模式启动游戏并把新功能跑一遍后：
- 游戏里任何地方调用 `t(key)`，如果发现 key 缺失，会自动把 key 写入 `tables/locale.csv`（zh/en 先留空）

你拿到更新后的 `tables/locale.csv`：
- 只需要把新出现的空白翻译补上

---

## 5. 你改完 CSV 后，怎么让游戏里生效？

你改的是 CSV（权威来源）。游戏运行时读的是 JSON（生成物）。

所以需要程序同学执行一次：

```bash
npm run csv:locale
```

它会用 `tables/locale.csv` 生成：
- `src/config/locales/zh.json`
- `src/config/locales/en.json`

然后重新运行/刷新游戏即可看到新文案。

---

## 6. 常见问题

### Q: 我填了翻译，但游戏里还是显示 key？
A: 通常是程序还没重新生成 JSON。让程序跑一次：`npm run csv:locale`。

### Q: key 看起来很乱，顺序不整齐怎么办？
A: 不影响功能。你可以直接在 Excel 里按 `key` 列排序查看；程序在 dev 自动补 key 时也会把 CSV 按前缀分组整理。
