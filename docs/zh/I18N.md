# i18n 本地化

> 权威来源：`tables/locale.csv`

---

## 技术部分

### 工作流程

```
tables/locale.csv  →  npm run csv -- locale  →  src/config/locales/zh.json / en.json
```

### 运行时

```typescript
import { t } from './data/i18n'

t('res.iron_ore')       // → '铁矿石'
t('nav.world')           // → '探索'
t('machine.furnace')     // → '冶炼炉'
t('recipe.smelt_iron')   // → '熔炼铁矿石'
```

### 自动补 key（开发模式）

dev 模式下，`t()` 遇到不存在的 key 会自动写入 `tables/locale.csv`（zh/en 留空），策划补翻译即可。

### 新增语言

1. `tables/locale.csv` 新增一列（如 `ja`）
2. `schemas/locale.schema.json` 的 `outputs` 加入输出路径
3. 运行 `npm run csv -- locale`
4. `src/data/i18n.ts` 注册语言代码

### key 前缀规范

| 前缀 | 用途 | 示例 |
|------|------|------|
| `res.*` | 资源名称 | `res.iron_ore` → '铁矿石' |
| `nav.*` | 导航标签 | `nav.world` → '探索' |
| `btn.*` | 按钮文本 | `btn.start` → '开始' |
| `machine.*` | 机器名称 | `machine.furnace` → '冶炼炉' |
| `recipe.*` | 配方名称 | `recipe.smelt_iron` → '熔炼铁矿石' |
| `task.*` | 任务描述 | `task.build_furnace` → '建造冶炼炉' |
| `chapter.*.name` | 章节名称 | `chapter.1.name` → '蛮荒时代' |
| `chapter.*.desc` | 章节描述 | `chapter.1.desc` → '...' |
| `tech.*.name` | 科技名称 | `tech.overclock.name` → '超频' |
| `tech.*.desc` | 科技描述 | `tech.overclock.desc` → '...' |
| `biome.*` | 群系名称 | `biome.plains` → '平原' |
| `world.node.*` | 探索节点名称 | `world.node.oak_tree` → '橡树' |

## 策划部分

### 翻译工作流程

1. 用 Excel 打开 `tables/locale.csv`（**保存时选 CSV UTF-8**）
2. 筛选 `zh` 或 `en` 列为空白的行
3. 补齐翻译（留空 = 暂时使用 key 本身作为回退）
4. 通知程序运行 `npm run csv -- locale`

### 已有 key 格式参考

| 列 | 内容示例 |
|----|---------|
| key | `res.iron_ore` |
| zh | `铁矿石` |
| en | `Iron Ore` |

### 添加新 key

1. 程序在代码中加入 `t('xxx.yyy')`
2. dev 模式下游戏跑一遍，key 自动进入 CSV
3. 策划补翻译
4. 程序运行 `npm run csv -- locale`

### 常见问题

**Q: 填了翻译但游戏里还是显示 key？**
A: 程序还没运行 `npm run csv -- locale` 生成 JSON。

**Q: CSV 打开乱码？**
A: 保存时选了普通 CSV，换成 **CSV UTF-8** 格式。
