## 中心化条件系统

> 核心文件：`src/data/conditions.ts`

所有"是否满足条件"的判断统一入口，各系统直接调用 `evaluateCondition(cond)`。

### 条件类型

| type | para 格式 | 说明 |
|---|---|---|
| `chapter` | 数字字符串，如 `'3'` | 当前章节 >= para |
| `task` | 数字字符串，如 `'10'` | 任务 ID 已完成 |
| `tech` | 科技 ID，如 `'basic_automation'` | 科技已研究 |
| `resource` | 资源 ID，如 `'weeds'` | 背包中拥有 > 0 |
| `tool` | `'toolType:level'`，如 `'stone_axe:2'` | 工具等级 >= level |

### 使用方式

```typescript
import { evaluateCondition } from './data/conditions'

// 单个条件判断
evaluateCondition({ type: 'chapter', para: '3' })  // → boolean
evaluateCondition({ type: 'resource', para: 'weeds' })
evaluateCondition(undefined)  // → true（无条件）

// 过滤可用项（直接用 def.showCond）
db.filter('machines', d => evaluateCondition(d.showCond))
```

### panel 条件

面板专用 `hideCond: 'panel'` 类型不走 `evaluateCondition`（需两轮 pass 避免循环依赖），由 `SideNav.vue` 单独处理。

## i18n 本地化

**权威来源**：`tables/locale.csv`（所有语言同一张表，列 = 语言代码）

**运行时**：生成到 `src/config/locales/zh.json` / `en.json`，由 `src/data/i18n.ts` 读取

```typescript
t('res.iron_ore')       // → '铁矿石'
t('nav.world')           // → '探索'
t('machine.furnace')     // → '冶炼炉'
```

### i18n key 前缀规范

| 前缀 | 用途 | 示例 |
|---|---|---|
| `res.*` | 资源名称 | `res.coal` |
| `nav.*` | 导航标签 | `nav.power` |
| `btn.*` | 按钮文本 | `btn.start` |
| `machine.*` | 机器名称 | `machine.furnace` |
| `recipe.*` | 配方名称 | `recipe.smelt_iron` |
| `task.*` | 任务描述 | `task.build_furnace` |
| `chapter.*.name/desc` | 章节名称/描述 | `chapter.1.name` |
| `tech.*.name/desc` | 科技名称/描述 | `tech.overclock.name` |
| `biome.*` | 群系名称 | `biome.plains` |
| `world.node.*` | 探索节点名称 | `world.node.oak_tree` |

### 新增 key 工作流

1. 在代码里写 `t('xxx.yyy')`（不存在会自动回退显示 key）
2. 打开 `tables/locale.csv`，对应行补翻译（zh / en 列留空会自动回退到 key）
3. 运行 `npm run csv -- locale` 生成 JSON

### 添加新语言

1. 在 `tables/locale.csv` 新增一列（如 `ja`）
2. 在 `schemas/locale.schema.json` 的 `outputs` 数组加入输出路径
3. 运行 `npm run csv -- locale`
4. 在 `src/data/i18n.ts` 注册语言代码

## 探索节点表 — biome_nodes.csv

| 列名 | 说明 |
|------|------|
| `id` | 节点唯一 ID |
| `locKey` | 节点名称 i18n key（悬浮提示） |
| `actionLocKey` | 按钮文字 i18n key |
| `type` | `click`（即时给资源）或 `timed`（倒计时后可领取） |
| `resourceId` | 产出资源 ID |
| `amount` | 每次产出数量 |
| `durationSec` | timed 节点倒计时秒数（仅 timed） |
| `posX` / `posY` | 面板内位置（0-100，百分比） |
| `hidden` | `true`=隐藏节点，需点击热区才显示按钮 |
| `hitArea` | SVG polygon 坐标（仅 hidden=true，如 `"30,50 45,45 55,52"`） |
| `requiredAbility` | 所需采集能力类型（如 `wood_ability`、`mining_ability`），能力不足无法采集 |
| `requiredAbilityValue` | 所需采集能力门槛值（玩家 ability >= 此值才能采集） |

**timed 节点流程**：`startTimedNode()` → 倒计时 → `done: true` → 用户点领取 → 给资源并清除状态（可重复触发）

**能力门槛逻辑**：
- 玩家 ability < requiredAbilityValue → 无法采集，显示提示
- 玩家 ability >= requiredAbilityValue → 可采集
- 超出门槛越多产出越多（公式：`amount += floor((ability - requiredAbilityValue) / 10)`）

## panelConfig.ts — 面板条件执行

位于 `src/data/panelConfig.ts`，面板条件复用 `src/data/conditions.ts` 的 `Condition` 类型（`panel` 类型仅用于 hideCond，由 SideNav 单独处理两轮 pass）：

```typescript
import { Condition, evaluateCondition } from './conditions'

export interface PanelShowCondition extends Condition {}
export interface PanelHideCondition { type: 'panel'; para: string }
export interface PanelDef { id, order, icon, labelKey, showCond?, hideCond? }

export function evaluateCondition(cond: Condition | undefined | null): boolean
```

**执行逻辑**（`SideNav.vue`）：
```typescript
function isVisible(def: PanelDef, visibleIds: Set<string>): boolean {
  // hideCond.panel 类型需两轮 pass
  if (def.hideCond?.type === 'panel' && visibleIds.has(def.hideCond.para)) return false
  // 通用条件走中心化 evaluateCondition
  if (def.showCond && !evaluateCondition(def.showCond)) return false
  return true
}
```

**默认激活面板**：App.vue 取 `order` 最小的可见面板，动态计算，不硬编码。

## worldStore — 探索系统接口

```typescript
clickNode(nodeId)       // 即时节点：给资源，返回获得数量（防连点）
startTimedNode(nodeId)   // 启动倒计时
claimTimedNode(nodeId)   // 领取：给资源 + 清除状态
revealNode(nodeId)       // 揭露隐藏节点
tick()                   // 每秒调用：更新 timed 节点 done 状态
```

**timed 节点不自动给资源**：tick() 只设 `done: true`，用户主动点领取才给资源，这样倒计时中途退出也不会丢进度。

## toolStore — 工具系统接口

```typescript
toolStore.getAbility('wood_ability')  // 获取当前砍伐能力值
toolStore.upgradeTool('axe')         // 升级斧子（消耗材料）
toolStore.levels['axe']              // 当前等级（0/1/2）
toolStore.isMaxLevel('axe')          // 是否已满级
toolStore.getLevelDef('axe')         // 当前等级定义（含属性值）
toolStore.getNextUpgradeCost('axe')   // 升下一级消耗
```

**工具与采集能力**：
- 工具提供 `wood_ability`（砍伐）和 `mining_ability`（采矿）属性
- 节点配置 `requiredAbility` + `requiredAbilityValue` 门槛
- 能力不足时 `clickNode` 返回 0，WorldPanel 显示提示
- 超出门槛时产出增加：`amount += floor((ability - threshold) / 10)`

## 添加新内容的通用流程

1. 确认对应 CSV（新建行或在现有 CSV 末尾追加）
2. 在 `tables/locale.csv` 补齐 i18n key
3. 运行 `npm run csv -- <表名>`（或 `npm run csv` 一键全部）
4. 如需注册组件（如新面板）：在 App.vue 添加，SidebarNav 条件会自动处理
