## 玩家游玩流程

### 第一阶段：初入蛮荒（纯手动采集）

**玩家行为**：点击地图节点，采集基础资源

**产出**：木头、石头、燧石

**体验目标**：让玩家理解"探索→采集→入库"的循环，建立基础认知

---

### 第二阶段：工具觉醒（制作工具）

**玩家行为**：用收集的素材制作工具

**产出**：石斧、石镐

**效果**：装备后每次点击采集量提升（石斧砍树+1，石镐挖矿+1）

**体验目标**：让玩家感受到"制作→变强"的正向反馈，开始规划资源用途

**里程碑**：制作出第一把石镐

---

### 第三阶段：矿石开采（资源积累）

**玩家行为**：用工具高效采集矿石资源

**产出**：铜矿石、锡矿石、煤炭

**体验目标**：工具效率提升后，采集从"艰难"变成"爽快"

**里程碑**：收集到足够的铜矿和锡矿

---

### 第四阶段：冶炼合金（第一次工业化）

**玩家行为**：建造熔炉，冶炼金属锭；建造合金炉，生产青铜锭

**产出**：铜锭、锡锭 → 青铜锭

**体验目标**：从纯手动采集，过渡到"半自动加工"

**里程碑**：产出第一块青铜锭 → 第一章结束

---

## 关键系统设计

### 1. 工具升级系统

#### 1.1 设计原则

- 工具预定义好了（石斧、石镐等）
- 工具初始为 0 级，0级不显示
- 达到显示条件后，工具才显示出来（1级可见）
- 玩家消耗材料进行升级：1级→2级→3级...
- 升级是永久的，不消耗工具
- 每级有独立的属性加成和材料消耗
- 工具等级越高，采集效率越高

#### 1.2 工具分类

| 类型 | 作用对象 | 示例 |
|------|----------|------|
| 采集工具 | 节点采集 | 石斧、石镐 |
| 加工工具 | 机器建造 | 锻造锤（制作机器时消耗） |

#### 1.3 工具等级与采集效率

| 工具 | 1级（木制） | 2级（石制） | 3级（第二章） |
|------|------------|------------|-------------|
| 石斧 | 2x | 3x | 4x |
| 石镐 | 2x | 3x | 4x |

**注：徒手 = 基准 1x，不算工具等级**

#### 1.4 升级消耗

| 工具 | 1级（木制）→ 2级（石制） |
|------|--------------------------|
| 石斧 | 木头×10 + 石头×5 |
| 石镐 | 木头×8 + 石头×8 + 燧石×4 |

**注：1级默认自带，不消耗材料；数值为初始估计，后续可调**

#### 1.5 升级系统数据结构

```
工具升级列表
├── stone_axe (石斧)
│   ├── levels:
│   │   ├── 1: { bonus: 2, cost: null }      // 1级默认自带
│   │   ├── 2: { bonus: 3, cost: { log_wood: 10, cobblestone: 5 } }  // 木→石
│   │   └── 3: { bonus: 4, cost: { ... } }
├── stone_pickaxe (石镐)
│   ├── levels:
│   │   ├── 1: { bonus: 2, cost: null }      // 1级默认自带
│   │   ├── 2: { bonus: 3, cost: { log_wood: 8, cobblestone: 8, flint: 4 } }  // 木→石
│   │   └── 3: { bonus: 4, cost: { ... } }
```

**注：显示条件后续实现，先默认都显示**

#### 1.6 采集判断逻辑

1. 玩家点击节点
2. 获取当前装备的采集工具等级
3. 检查 `玩家工具等级 >= 节点要求等级`
4. 不满足 → 提示"需要XX工具（等级X）"
5. 满足 → 获得 `节点基础产出 × 工具效率倍率` 资源

### 2. 节点条件系统

#### 2.1 设计原则

- 每个节点可设置工具等级门槛
- 玩家必须达到门槛才能采集该节点
- 低于门槛时提示"需要XX工具"
- 节点找到即可采集（无需刷新）

#### 2.2 节点数据结构

```
节点ID
├── 位置 (posX, posY)
├── 资源类型 (resourceId)
├── 基础产出量 (baseAmount)
├── 工具等级要求 (requiredToolLevel)
├── 是否隐藏 (hidden)
└── 采集动画/音效
```

#### 2.3 第一章节点配置

| 节点ID | 资源 | 基础产出 | 工具要求 | 隐藏 |
|--------|------|---------|---------|------|
| oak_tree_1 | log_wood | 1 | 0（徒手） | 否 |
| stone_rock_1 | cobblestone | 1 | 0（徒手） | 否 |
| flint_patch_1 | flint | 1 | 0（徒手） | 否 |
| copper_ore_1 | copper_ore | 1 | 1（石镐） | 否 |
| tin_ore_1 | tin_ore | 1 | 1（石镐） | 否 |
| coal_ore_1 | coal | 1 | 1（石镐） | 否 |
| iron_ore_rich_1 | iron_ore | 10 | 1（石镐） | 是 |
| copper_ore_rich_1 | copper_ore | 5 | 1（石镐） | 是 |

#### 2.4 隐藏节点揭示方式

**后续再设计（引导系统的一部分）**
- 可能是：在地图上走/探索，自动揭示
- 可能是：完成某些任务/条件后揭示

**暂定揭示条件**：
- 玩家在附近区域有一定采集次数后揭示

### 4. 探索节点系统

**基础规则**：
- 节点找到后即可采集，无次数限制
- 点击节点获取资源，手动采集

**采集流程**：
1. 玩家点击节点
2. 检查工具等级是否满足 `requiredToolLevel`
3. 不满足 → 提示"需要XX工具"
4. 满足 → 获得 `baseAmount × 工具效率倍率` 资源

## 方案总结

### 工具升级系统
- 工具预定义，初始 0 级
- 消耗材料升级：0级→1级→2级...
- 每级有属性加成和消耗材料，可配置
- 工具等级决定采集效率倍率

### 节点条件系统
- 节点设置 `requiredToolLevel` 字段
- 玩家工具等级 >= 节点要求 → 可采
- 低于门槛 → 提示"需要XX工具"
- 产出量 = 基础产出 × 工具效率倍率

### 隐藏节点
- 初始隐藏，找到后揭示
- 通常产量更高
- 揭示方式后续设计（引导系统）

## 讨论记录

1. 探索节点无限可点击，只是手点很慢
2. 工具是升级系统，预定义，消耗材料升级，不消耗
3. 采集点需要工具等级条件
4. 青铜是直接用铜锭+锡锭在合金炉熔炼，不需要青铜矿石
5. 青铜配方：3铜:1锡
6. 铜矿/锡矿/煤矿：石镐（1级）即可采集
7. 隐藏节点：可以藏富集矿、特殊矿，产量高但初始隐藏
8. 燧石不一定是隐藏的，可以是普通节点
9. 引导后续再做
10. 工具是升级系统，不是制作系统
11. 0级不显示，1级起才显示
12. 工具需要显示条件，满足条件后才显示
13. 石斧：1级木制、2级石制
14. 石镐：1级木制、2级石制
15. 显示条件后续实现
